Design the high-level architecture for the tutoring platform MVP.
The system must support:

A student creating an account and completing an intake quiz
That quiz output being used to build an initial learning profile

A student opening a tutoring session and having a Socratic conversation with an AI

The AI using the student's learning profile to calibrate every response
After the session ends, the system extracting only new information from the conversation and updating the profile — not rewriting it wholesale

Your job:

List every major system component you think exists
For each component, state what it does in one sentence
Draw the data flow — in plain English or pseudocode, not a diagram tool yet — from the moment a student hits "start session" to the moment their profile is updated after the session ends
Identify where you think the hardest architectural problems are
